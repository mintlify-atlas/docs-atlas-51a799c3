This guide shows you how to integrate WideLogger with Express to automatically log HTTP requests with rich contextual data.

## Prerequisites

Install WideLogger and Express:

```bash
npm install widelogger express
```

## Integration steps

<Steps>

<Step title="Initialize the logger">

Create a logger instance with your service configuration:

```typescript logger.ts
import { widelogger } from "widelogger";

export const { widelog, destroy } = widelogger({
  service: "example-api",
  defaultEventName: "http_request",
  version: "1.0.0",
  commitHash: process.env.COMMIT_SHA,
});
```

</Step>

<Step title="Create logging middleware">

Build middleware that wraps each request in a logging context:

```typescript middleware/logging.ts
import type { NextFunction, Request, Response } from "express";
import { widelog } from "../logger";

export const logging = (
  request: Request,
  response: Response,
  next: NextFunction
) => {
  widelog.context(
    () =>
      new Promise<void>((resolve) => {
        widelog.set("method", request.method);
        widelog.set("path", request.path);
        widelog.time.start("duration_ms");

        response.on("finish", () => {
          widelog.set("status_code", response.statusCode);
          widelog.time.stop("duration_ms");
          widelog.flush();
          resolve();
        });

        next();
      })
  );
};
```

The middleware:
- Creates an isolated logging context for each request
- Captures method, path, and timing information
- Flushes the log when the response finishes
- Uses a Promise to maintain the context throughout the request lifecycle

</Step>

<Step title="Register middleware and routes">

Apply the logging middleware globally and set up your routes:

```typescript server.ts
import express from "express";
import { destroy } from "./logger";
import { logging } from "./middleware/logging";
import { checkout } from "./routes/checkout";
import { health } from "./routes/health";

const app = express();

app.use(express.json());
app.use(logging);

app.get("/health", health);
app.post("/checkout", checkout);

const server = app.listen(3000, () => {
  console.log("Listening on http://localhost:3000");
});

process.on("SIGINT", async () => {
  server.close();
  await destroy();
  process.exit(0);
});
```

<Tip>
Always call `destroy()` during graceful shutdown to ensure all logs are flushed before the process exits.
</Tip>

</Step>

<Step title="Add fields in route handlers">

Enrich logs with request-specific data in your handlers:

```typescript routes/checkout.ts
import type { Request, Response } from "express";
import { widelog } from "../logger";

export const checkout = (request: Request, response: Response) => {
  const { userId } = request.body;

  widelog.set("user.id", userId);
  widelog.set("user.plan", "premium");

  widelog.time.start("db_ms");
  // const order = await processOrder(userId);
  const order = { id: "ord_001", totalCents: 14_999, itemCount: 3 };
  widelog.time.stop("db_ms");

  widelog.set("order.id", order.id);
  widelog.set("order.total_cents", order.totalCents);
  widelog.count("order.items", order.itemCount);

  response.json({ orderId: order.id });
};
```

All fields added within the same request automatically belong to the same wide event—no need to manually pass context between middleware and handlers.

</Step>

</Steps>

## Best practices

<AccordionGroup>
  <Accordion title="Use response.on('finish') for cleanup">
    Express doesn't have a built-in lifecycle hook that runs after the response is sent. Listen to the `finish` event on the response object to ensure logs are flushed only after the response is complete.
  </Accordion>
  
  <Accordion title="Wrap context in a Promise">
    The middleware uses `widelog.context()` with a Promise to maintain async context throughout the request lifecycle. This ensures all log operations within the request share the same context.
  </Accordion>
  
  <Accordion title="Handle graceful shutdown">
    Always register a SIGINT handler that calls `destroy()` to flush any pending logs before the process exits.
  </Accordion>
</AccordionGroup>
