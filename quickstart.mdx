This guide walks you through creating a complete Express API with WideLogger integration. You'll learn the core concepts by building a working example.

## Setup your project

<Steps>
  <Step title="Create a new project">
    Initialize a new Node.js project:
    ```bash
    mkdir my-api && cd my-api
    npm init -y
    ```

    Set the module type in `package.json`:
    ```json package.json
    {
      "type": "module"
    }
    ```
  </Step>

  <Step title="Install dependencies">
    Install WideLogger and Express:
    ```bash
    npm install widelogger express
    npm install -D @types/express typescript
    ```
  </Step>
</Steps>

## Create the logger

Create a shared logger instance that you'll import throughout your application:

```ts src/logger.ts
import { widelogger } from "widelogger";

export const { widelog, destroy } = widelogger({
  service: "example-api",
  defaultEventName: "http_request",
  version: "1.0.0",
  commitHash: process.env.COMMIT_SHA,
});
```

<Tip>
  The `widelogger()` function returns both `widelog` (for logging operations) and `destroy()` (for graceful shutdown).
</Tip>

## Build logging middleware

Create middleware that wraps each request in an isolated context:

```ts src/middleware/logging.ts
import type { Request, Response, NextFunction } from "express";
import { widelog } from "../logger";

export const logging = (
  request: Request,
  response: Response,
  next: NextFunction
) => {
  widelog.context(
    () =>
      new Promise<void>((resolve) => {
        // Capture request details
        widelog.set("method", request.method);
        widelog.set("path", request.path);
        widelog.time.start("duration_ms");

        // Wait for response to finish
        response.on("finish", () => {
          widelog.set("status_code", response.statusCode);
          widelog.time.stop("duration_ms");
          widelog.flush(); // Emit the event
          resolve();
        });

        next();
      })
  );
};
```

<Note>
  `widelog.context()` creates an isolated async context using AsyncLocalStorage. All operations inside this context are automatically tracked, even across async boundaries.
</Note>

## Create a route handler

Build a route that accumulates additional context:

```ts src/routes/checkout.ts
import type { Request, Response } from "express";
import { widelog } from "../logger";

export const checkout = (request: Request, response: Response) => {
  const { userId } = request.body;

  // Add user context
  widelog.set("user.id", userId);
  widelog.set("user.plan", "premium");

  // Track database timing
  widelog.time.start("db_ms");
  const order = { id: "ord_001", totalCents: 14_999, itemCount: 3 };
  widelog.time.stop("db_ms");

  // Add order details
  widelog.set("order.id", order.id);
  widelog.set("order.total_cents", order.totalCents);
  widelog.count("order.items", order.itemCount);

  response.json({ orderId: order.id });
};
```

<Tip>
  Notice how the route handler doesn't need to know about context creation or flushing — it just adds fields. The middleware handles the rest.
</Tip>

## Wire it together

Create the Express server with graceful shutdown:

```ts src/server.ts
import express from "express";
import { destroy } from "./logger";
import { logging } from "./middleware/logging";
import { checkout } from "./routes/checkout";

const app = express();

app.use(express.json());
app.use(logging);

app.post("/checkout", checkout);

const server = app.listen(3000, () => {
  console.log("Listening on http://localhost:3000");
});

// Graceful shutdown
process.on("SIGINT", async () => {
  server.close();
  await destroy();
  process.exit(0);
});
```

## Key concepts

<CardGroup cols={2}>
  <Card title="Context isolation" icon="shield">
    `widelog.context()` creates an isolated async context for each request using AsyncLocalStorage
  </Card>
  <Card title="Dot notation" icon="circle-nodes">
    Keys like `"user.id"` automatically expand to nested objects: `{ user: { id: "..." } }`
  </Card>
  <Card title="Accumulation" icon="layer-group">
    Multiple operations on the same key are merged intelligently (last write wins for `set`, sum for `count`, etc.)
  </Card>
  <Card title="Automatic routing" icon="route">
    Events with `status_code >= 500` or `outcome === "error"` log at error level
  </Card>
</CardGroup>

## Available operations

WideLogger provides multiple ways to accumulate context:

| Method | Description | Example |
|--------|-------------|----------|
| `set(key, value)` | Set a field (last write wins) | `widelog.set("user.id", "123")` |
| `count(key, amount?)` | Increment a counter | `widelog.count("items", 3)` |
| `append(key, value)` | Append to an array | `widelog.append("tags", "premium")` |
| `max(key, value)` | Track maximum value | `widelog.max("peak_memory", 512)` |
| `min(key, value)` | Track minimum value | `widelog.min("min_latency", 10)` |
| `time.start(key)` | Start a timer | `widelog.time.start("db_ms")` |
| `time.stop(key)` | Stop timer and record ms | `widelog.time.stop("db_ms")` |
| `errorFields(error)` | Extract error details | `widelog.errorFields(err)` |

## Error handling

Handle errors gracefully and capture error context:

```ts
app.post("/checkout", async (request, response) => {
  try {
    const order = await processOrder(request.body.userId);
    widelog.set("outcome", "success");
    response.json({ orderId: order.id });
  } catch (error) {
    widelog.set("outcome", "error");
    widelog.errorFields(error); // Adds error_name, error_message, error_stack
    response.status(500).json({ error: "Checkout failed" });
  }
});
```

<Note>
  `errorFields()` automatically extracts `error_name`, `error_message`, and `error_stack` from Error objects.
</Note>
