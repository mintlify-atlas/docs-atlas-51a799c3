## Overview

WideLogger is a structured logging SDK for Node.js that transforms how you capture application events. Instead of scattering multiple log statements throughout your code, WideLogger lets you accumulate context as your request flows through your application, then emit a single comprehensive event at the end.

Built on [Pino](https://github.com/pinojs/pino) and `AsyncLocalStorage`, WideLogger provides zero-leak context isolation across concurrent requests — perfect for modern async Node.js applications.

## Why wide events?

Traditional logging creates multiple log entries per request, making it difficult to understand the full story. Wide event logging, inspired by [loggingsucks.com](https://loggingsucks.com), accumulates all relevant context into a single structured event:

```json
{
  "event_name": "http_request",
  "method": "POST",
  "path": "/checkout",
  "user": {
    "id": "usr_123",
    "plan": "premium"
  },
  "order": {
    "id": "ord_001",
    "total_cents": 14999,
    "items": 3
  },
  "db_ms": 42,
  "duration_ms": 156,
  "status_code": 200
}
```

One request, one event, complete story.

## How it works

<Steps>
  <Step title="Create a logger instance">
    Initialize WideLogger with your service metadata
    ```ts
    const { widelog, destroy } = widelogger({
      service: "checkout-api",
      defaultEventName: "http_request"
    });
    ```
  </Step>
  <Step title="Wrap requests in context">
    Use middleware to create isolated context for each request
    ```ts
    app.use((req, res, next) => {
      widelog.context(async () => {
        // Your request handling
      });
    });
    ```
  </Step>
  <Step title="Accumulate fields anywhere">
    Add context from route handlers, services, or utilities
    ```ts
    widelog.set("user.id", userId);
    widelog.time.start("db_ms");
    widelog.count("items", 3);
    ```
  </Step>
  <Step title="Flush the event">
    Emit the complete structured event when the request finishes
    ```ts
    widelog.flush();
    ```
  </Step>
</Steps>
