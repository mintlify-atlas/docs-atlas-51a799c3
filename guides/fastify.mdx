This guide shows you how to integrate WideLogger with Fastify using the plugin system for clean, modular request logging.

## Prerequisites

Install WideLogger, Fastify, and fastify-plugin:

```bash
npm install widelogger fastify fastify-plugin
```

## Integration steps

<Steps>

<Step title="Initialize the logger">

Create a logger instance with your service configuration:

```typescript logger.ts
import { widelogger } from "widelogger";

export const { widelog, destroy } = widelogger({
  service: "example-api",
  defaultEventName: "http_request",
  version: "1.0.0",
  commitHash: process.env.COMMIT_SHA,
});
```

</Step>

<Step title="Create a logging plugin">

Build a Fastify plugin that uses the `onRequest` hook to wrap each request in a logging context:

```typescript plugins/logging.ts
import type { FastifyPluginCallback } from "fastify";
import fp from "fastify-plugin";
import { widelog } from "../logger";

const loggingPlugin: FastifyPluginCallback = (app, _options, pluginDone) => {
  app.addHook("onRequest", (request, reply, done) => {
    widelog.context(
      () =>
        new Promise<void>((resolve) => {
          widelog.set("method", request.method);
          widelog.set("path", request.url);
          widelog.time.start("duration_ms");

          reply.raw.on("finish", () => {
            widelog.set("status_code", reply.statusCode);
            widelog.time.stop("duration_ms");
            widelog.flush();
            resolve();
          });

          done();
        })
    );
  });

  pluginDone();
};

export const logging = fp(loggingPlugin);
```

The plugin:
- Uses `onRequest` hook to intercept all incoming requests
- Creates an isolated context with `widelog.context()`
- Listens to `reply.raw.on('finish')` to flush logs after response completes
- Wraps logic in a Promise to maintain async context
- Uses `fastify-plugin` to prevent plugin encapsulation

</Step>

<Step title="Register plugin and routes">

Register the logging plugin globally and set up your route plugins:

```typescript server.ts
import fastify from "fastify";
import { destroy } from "./logger";
import { logging } from "./plugins/logging";
import { checkoutRoutes } from "./routes/checkout";
import { healthRoutes } from "./routes/health";

const app = fastify();

app.register(logging);
app.register(healthRoutes);
app.register(checkoutRoutes);

app.listen({ port: 3000 }, () => {
  console.log("Listening on http://localhost:3000");
});

process.on("SIGINT", async () => {
  await app.close();
  await destroy();
  process.exit(0);
});
```

<Tip>
Call `app.close()` before `destroy()` during shutdown to gracefully close all connections before flushing logs.
</Tip>

</Step>

<Step title="Add fields in route handlers">

Define routes as plugins and enrich logs with request-specific data:

```typescript routes/checkout.ts
import type { FastifyPluginCallback } from "fastify";
import { widelog } from "../logger";

interface CheckoutBody {
  userId: string;
}

export const checkoutRoutes: FastifyPluginCallback = (app, _options, done) => {
  app.post<{ Body: CheckoutBody }>("/checkout", (request) => {
    const { userId } = request.body;

    widelog.set("user.id", userId);
    widelog.set("user.plan", "premium");

    widelog.time.start("db_ms");
    // const order = await processOrder(userId);
    const order = { id: "ord_001", totalCents: 14_999, itemCount: 3 };
    widelog.time.stop("db_ms");

    widelog.set("order.id", order.id);
    widelog.set("order.total_cents", order.totalCents);
    widelog.count("order.items", order.itemCount);

    return { orderId: order.id };
  });

  done();
};
```

All fields added within the request context automatically merge into the same wide event.

</Step>

</Steps>

## Best practices

<AccordionGroup>
  <Accordion title="Use fastify-plugin for global plugins">
    Wrap your logging plugin with `fastify-plugin` (imported as `fp`) to prevent encapsulation. This ensures the logging context is available across all routes, including those registered by other plugins.
  </Accordion>
  
  <Accordion title="Listen to reply.raw.on('finish')">
    Fastify's lifecycle hooks don't include a post-response hook. Use `reply.raw.on('finish')` to flush logs only after the response is fully sent.
  </Accordion>
  
  <Accordion title="Structure routes as plugins">
    Organize routes as Fastify plugins for better modularity and easier testing. This pattern scales well as your application grows.
  </Accordion>
  
  <Accordion title="Close app before destroying logger">
    Use `app.close()` in your shutdown handler to gracefully close Fastify before calling `destroy()` to flush remaining logs.
  </Accordion>
</AccordionGroup>
