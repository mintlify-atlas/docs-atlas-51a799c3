This guide demonstrates how to integrate WideLogger with Bun's native HTTP server using `Bun.serve()` for fast, structured request logging.

## Prerequisites

Install WideLogger:

```bash
bun add widelogger
```

## Integration steps

<Steps>

<Step title="Initialize the logger">

Create a logger instance with your service configuration:

```typescript logger.ts
import { widelogger } from "widelogger";

export const { widelog, destroy } = widelogger({
  service: "example-api",
  defaultEventName: "http_request",
  version: "1.0.0",
  commitHash: process.env.COMMIT_SHA,
});
```

</Step>

<Step title="Wrap fetch handler with logging context">

Bun's server uses a `fetch` function to handle requests. Wrap this function with `widelog.context()` to create isolated logging contexts:

```typescript server.ts
import { serve } from "bun";
import { destroy, widelog } from "./logger";
import { checkout } from "./routes/checkout";
import { health } from "./routes/health";

const routes: Record<
  string,
  (request: Request) => Response | Promise<Response>
> = {
  "POST /checkout": checkout,
  "GET /health": health,
};

const server = serve({
  port: 3000,
  fetch: (request) =>
    widelog.context(async () => {
      const url = new URL(request.url);
      const route = routes[`${request.method} ${url.pathname}`];

      widelog.set("method", request.method);
      widelog.set("path", url.pathname);
      widelog.time.start("duration_ms");

      try {
        if (!route) {
          widelog.set("status_code", 404);
          widelog.set("outcome", "not_found");
          return new Response("Not Found", { status: 404 });
        }

        const response = await route(request);
        widelog.set("status_code", response.status);
        widelog.set("outcome", "success");
        return response;
      } catch (error) {
        widelog.set("status_code", 500);
        widelog.set("outcome", "error");
        widelog.errorFields(error);
        return Response.json(
          { error: "internal server error" },
          { status: 500 }
        );
      } finally {
        widelog.time.stop("duration_ms");
        widelog.flush();
      }
    }),
});

console.log(`Listening on http://localhost:${server.port}`);

process.on("SIGINT", async () => {
  server.stop();
  await destroy();
  process.exit(0);
});
```

Key aspects:
- Each request is wrapped in `widelog.context()` for isolation
- Basic routing is handled with a simple lookup table
- Error handling adds error fields using `errorFields()`
- `finally` block ensures timing stops and logs flush on every request
- Graceful shutdown stops the server and destroys the logger

</Step>

<Step title="Add fields in route handlers">

Route handlers can add fields that automatically merge into the same wide event:

```typescript routes/checkout.ts
import { widelog } from "../logger";

interface CheckoutBody {
  userId: string;
}

export const checkout = async (request: Request) => {
  const body = await request.json<CheckoutBody>();

  // These fields are added to the same wide event that the
  // server middleware started — no need to pass context around.
  widelog.set("user.id", body.userId);
  widelog.set("user.plan", "premium");

  widelog.time.start("db_ms");
  // const order = await processOrder(body.userId);
  const order = { id: "ord_001", totalCents: 14_999, itemCount: 3 };
  widelog.time.stop("db_ms");

  widelog.set("order.id", order.id);
  widelog.set("order.total_cents", order.totalCents);
  widelog.count("order.items", order.itemCount);

  return Response.json({ orderId: order.id });
};
```

Context propagation happens automatically—any `widelog` calls within the context share the same event.

</Step>

</Steps>

## Best practices

<AccordionGroup>
  <Accordion title="Use try-catch-finally for reliability">
    Always use a `finally` block to stop timers and flush logs. This ensures logs are emitted even when errors occur.
  </Accordion>
  
  <Accordion title="Add outcome fields for observability">
    Track request outcomes (`success`, `not_found`, `error`) to make it easy to filter and aggregate logs by result type.
  </Accordion>
  
  <Accordion title="Use errorFields() for structured errors">
    When catching errors, use `widelog.errorFields(error)` to extract structured error information (message, stack, name) into your wide event.
  </Accordion>
  
  <Accordion title="Stop server before destroying logger">
    During shutdown, call `server.stop()` before `destroy()` to ensure no new requests arrive while flushing logs.
  </Accordion>
</AccordionGroup>
